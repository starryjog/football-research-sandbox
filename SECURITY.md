# Security Policy

更新时间：2026-06-28

本仓库主要维护公开足球研究资料和静态站点代码。安全与风险报告重点包括敏感信息误收录、未成年人隐私、恶意链接、供应链风险和页面脚本问题。

## 请报告的问题

- 非公开个人信息、联系方式、家庭住址、证件信息或私人社媒内容被收录。
- 青年球员资料存在明显隐私、骚扰或误导风险。
- 来源链接指向恶意软件、钓鱼页面或已被劫持的网站。
- 页面脚本、依赖或 GitHub Actions 存在可利用风险。
- 公开资料被错误解释为官方、全量或实时数据库，可能造成重大误导。

普通数据错误、死链和口径争议可以直接开 issue；涉及隐私或安全风险时，请不要在公开 issue 中贴敏感细节。

## 报告方式

如果仓库启用了 GitHub private vulnerability reporting，请优先使用该入口。否则请联系仓库维护者，并只提供定位问题所需的最小信息：

- 问题类型。
- 涉及文件、页面或链接。
- 为什么有隐私或安全风险。
- 建议的最小修复方式。

## 处理原则

- 优先移除或降级敏感、不可公开验证或可能造成伤害的内容。
- 保留必要的审计说明，但不重复传播敏感材料。
- 第三方来源存在风险时，先删除或替换链接，再补核验备注。
- 数据口径变更同步更新 `docs/scope.md`、`docs/source-policy.md` 或 `CHANGELOG.md`。
